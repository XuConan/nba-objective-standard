# NBA 客观数据标准评选 Skill

## 简介

本 Skill 提供一套基于统一客观数据标准的 NBA 奖项评选方案，完全排除媒体叙事、印象流和人气因素。适用于支持自定义 Markdown Skill 的平台（如 Claude、Cursor、自定义 GPT 等）。

## 核心原则

- **数据优先**：结果基于 Basketball-Reference、NBA 官方 API 等真实统计数据
- **以赢为主**：常规赛个人奖项中，球队战绩占 60% 权重
- **标准统一**：同一赛季所有球员使用相同的评分模型
- **可重复验证**：所有公式公开，允许任何人基于相同数据复现

## 覆盖奖项

### 常规赛
- 数据王：得分、篮板、助攻、抢断、盖帽（纯数据排名）
- MVP（最有价值球员）
- DPOY（最佳防守球员）
- 最佳阵容（一阵、二阵、三阵）
- 最佳防守阵容（一防、二防）
- 最佳新秀、最快进步球员、最佳第六人

### 季后赛
- 总冠军
- FMVP（总决赛最有价值球员）
- **季后赛 MVP**（整个季后赛表现，新增奖项）
- **冠军含金量指数 CQI**（可选，评估冠军实力）

## 评分规则速查

| 奖项 | 战绩权重 | 数据权重 | 核心指标 |
|------|----------|----------|----------|
| MVP | 60% | 40% | PER, WS/48, VORP |
| DPOY | 无关 | 100% | DBPM, DWS, 抢断+盖帽 |
| FMVP | 60%（总决赛胜负） | 40% | 总决赛 PER, WS/48 |
| 季后赛 MVP | 50%（最终轮次） | 50% | 季后赛 PER, WS/48, VORP |
| CQI | 综合公式 | — | 自身统治力+对手强度+伤病+传奇性 |

## 快速使用

**触发词**（在对话中输入以下任一）：

/客观数据标准
/statStandard

后跟具体指令，例如：

- `评选 2001 年全部奖项`
- `季后赛MVP 2023`
- `计算 2016 年总冠军含金量`

AI 将自动按规则输出报告，包含数据源声明、评分过程和最终结果。

## 数据源

Skill 会自动尝试以下数据源并交叉验证（优先级从高到低）：

1. **Basketball-Reference** – 高阶数据最全
2. **nba_api** – 官方 API，数据规范
3. **balldontlie API** – 免费备选
4. 虎扑 / 新浪 – 中文基础数据（辅助）

若某源不可用，自动降级并注明置信度。缺失高阶数据（如早期 VORP）会用 PER+WS/48 加权估算。

## 输出示例

完整报告包含：
- 数据源声明及一致程度
- 常规赛数据王（得分、篮板等）
- MVP 前 5 名得分表格
- 最佳阵容一阵/二阵/三阵
- DPOY 及最佳防守阵容
- 季后赛总冠军、FMVP、季后赛 MVP
- 冠军含金量指数（若用户要求）

## 常见问题

**Q：为什么季后赛 MVP 和 FMVP 分开？**  
A：FMVP 只看总决赛（最多 7 场），偶然性大；季后赛 MVP 覆盖整个季后赛，更全面。

**Q：战绩分如何计算？**  
A：常规赛按联盟总排名线性赋分（第 1 名 60 分，第 20 名 1 分）。季后赛按最终轮次赋分（冠军 100，亚军 80，分区决赛 60，次轮 40，首轮 20）。

**Q：早期赛季数据缺失怎么办？**  
A：Skill 会自动降级并用已有指标加权估算，同时标注 `*`。

**Q：可以分析进行中的赛季吗？**  
A：可以，但数据需在赛季结束约一个月后才完全稳定。未完成赛季的结果仅供参考。
